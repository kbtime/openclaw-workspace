## 二、技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| **知识库仓库** | GitHub 私有仓库 | 版本控制、同步 |
| **向量 API** | SiliconFlow | 文档向量化、语义检索 |
| **嵌入模型** | Qwen3-Embedding-8B | 生成文本向量 |
| **重排序模型** | Qwen3-Reranker-8B | 优化检索结果排序 |
| **子 Agent** | 阿里云百炼 qwen3.5-flash | 知识库问答、合同审核 |
| **主 Agent** | GLM-5 | 综合分析、决策执行 |

## 三、用户端使用流程

### 步骤 1：克隆知识库仓库

```bash
# 在你的电脑上
git clone https://github.com/kbtime/knowledge-base.git
cd knowledge-base
```

### 步骤 2：添加知识库文档

将你的文档放入对应目录：

```
knowledge-base/
├── contracts/          # 合同模板
│   ├── 劳动合同模板.md
│   ├── 采购合同模板.md
│   └── 保密协议模板.md
├── policies/           # 公司制度
│   ├── 报销制度.md
│   ├── 请假制度.md
│   ├── 采购制度.md
│   └── 差旅制度.md
├── templates/          # 审批模板
│   └── 付款申请模板.md
└── checklists/         # 检查清单
    └── 合同审核清单.md
```

### 步骤 3：提交并推送

```bash
git add .
git commit -m "添加新制度文档"
git push
```

### 步骤 4：通知 VPS 同步

在对话中告诉我：
> "知识库已更新，帮我同步"

或者我可以设置自动定时同步。

## 四、VPS 端处理流程

### 自动同步（推荐）

我可以设置定时任务，每小时自动拉取并索引：

```bash
# 查看当前同步状态
cd ~/.openclaw/workspace/knowledge-base
git pull

# 重新索引
node ~/.openclaw/workspace/skills/knowledge-base/scripts/agent.js --index
```

### 手动同步

你随时可以让我执行同步。

## 五、查询使用方式

### 方式 1：对话中直接问

```
你: 按照公司制度，报销 5000 元需要谁审批？
我: [自动调用知识库查询] 根据《报销制度》，5000 元以上需要总经理审批。
```

### 方式 2：合同审核

```
你: 帮我审核这份合同 [上传合同文件]
我: [调用知识库对比制度] 发现以下问题：
    1. 付款条款不符合公司规定
    2. 缺少违约责任条款
    建议：...
```

### 方式 3：制度查询

```
你: 公司年假政策是什么？
我: [检索知识库] 根据《请假制度》：
    - 工龄 1-5 年：5 天
    - 工龄 5-10 年：10 天
    - 工龄 10 年以上：15 天
```

## 六、文档编写规范

为了让检索更准确，建议按以下规范编写：

### 1. 文件命名

```
✅ 好：报销制度.md、劳动合同模板.md
❌ 差：1.md、新建文档.md
```

### 2. 标题结构

```markdown
# 制度名称

## 适用范围
...

## 具体规定

### 金额限制
| 金额 | 审批人 |
|------|-------|
| < 500 | 直属主管 |
| > 5000 | 总经理 |

## 注意事项
...
```

### 3. 关键信息突出

```markdown
✅ 重要条款用粗体或列表
**付款期限：30 天内**

✅ 金额、日期等关键数据用表格
✅ 审批流程用步骤编号
```

## 八、配置信息汇总

### Git 仓库

| 配置项 | 值 |
|-------|-----|
| 仓库地址 | `https://github.com/kbtime/knowledge-base` |
| 用户名 | `kbtime` |
| Token | 从环境变量 `GITHUB_TOKEN` 读取 |

### 向量 API (SiliconFlow)

| 配置项 | 值 |
|-------|-----|
| API URL | `https://api.siliconflow.cn` |
| API Key | 从环境变量 `SILICONFLOW_API_KEY` 读取 |
| 嵌入模型 | `Qwen/Qwen3-Embedding-8B` |
| 重排序模型 | `Qwen/Qwen3-Reranker-8B` |

### 子 Agent (阿里云百炼)

| 配置项 | 值 |
|-------|-----|
| 模型 | `qwen3.5-flash` |
| API Key | 从环境变量 `DASHSCOPE_API_KEY` 读取 |
